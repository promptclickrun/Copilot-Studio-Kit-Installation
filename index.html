Overview & Features

▼

The Copilot Studio Kit is a free, open-source comprehensive toolset developed by Microsoft. Updated monthly, it provides a full enterprise governance, testing, and monitoring platform for Copilot Studio agents.

GitHub Repository: https://github.com/microsoft/Power-CAT-Copilot-Studio-Kit

Key Capabilities

🧪 Test Automation

Batch-test agents with configurable test sets. Supports Response Match, Topic Match, Multi-Turn, Attachment Match, and AI-graded Generative Answers via AI Builder rubrics.

🗺 Agent Inventory

Tenant-wide view of all Copilot Studio agents: auth mode, knowledge sources, orchestration type, usage stats.

✅ Compliance Hub

Continuously evaluate agents against risk thresholds. Auto-creates compliance cases with SLA-driven workflows, Teams/Outlook notifications.

📊 Conversation KPIs

Aggregated conversation-level KPIs in Dataverse with embedded Power BI dashboard for trend analysis and agent performance tracking.

💡 Prompt Advisor

AI-assisted prompt engineering with confidence scoring and suggested refinements. Each iteration ~120 AI Builder credits.

📋 Rubric Refinement

Create and iterate on reusable AI grading rubrics to align automated evaluation with human judgment and org quality standards.

🔍 Conversation Analyzer

Analyze agent conversations using custom prompts for deeper operational insights beyond built-in analytics.

📂 SharePoint Sync

Scheduled, selective content synchronization from SharePoint sites to Copilot Studio knowledge bases.

🔧 Agent Review Tool

Solution-level analysis identifying anti-patterns, security issues, and performance risks in agent configurations.

💬 Webchat Playground

Visual webchat customization with live preview and automatic HTML generation for embedding agents.

🎨 Adaptive Cards Gallery

Pre-built adaptive card templates with dynamic data binding examples for agent responses.

🚀 Pipeline Automation

Quality-gate agent deployments via Power Platform Pipelines — only agents passing test thresholds deploy to production.

📅
Release Cadence The kit is updated approximately monthly. The latest release (April 2025+) added multi-turn test automation support, new string comparison operators, Agent Review Tool, and Agent Inventory. Subscribe to GitHub releases to receive notifications.

Prerequisites

▼
⚠
Complete Before Installing Validate every prerequisite below before attempting installation. Missing dependencies — especially the Creator Kit and PCF — are the #1 cause of failed installs.

Mandatory Requirements

Required
Power Platform Environment + Dataverse
Dataverse must be enabled as the data store in your target environment.
Required
System Administrator Role
Installer must have the System Administrator security role in the target environment.
Required
Power Apps Model-Driven License
Adequate licensing to run a Power Apps model-driven application.
Required
Power Automate Premium License
Licensing to run Power Automate cloud flows with Premium connectors.
Required — Enable First!
PCF Components Enabled
Power Apps Component Framework for canvas apps must be ON in the Dataverse environment settings before any solution import.
💡
How to Enable PCF Power Platform Admin Center → Environments → [Your Environment] → Settings → Features → Enable "Power Apps component framework for canvas apps" toggle.

Required Dependencies (Install First!)

🚫
Critical Dependency Creator Kit Core must be installed BEFORE the Copilot Studio Kit. Installing CSK without Creator Kit will trigger a dependency error that stops the import. There is no workaround — Creator Kit must be present first.
1

Download Creator Kit Core

Navigate to: https://github.com/microsoft/powercat-creator-kit/releases

Download the latest CreatorKitCore_managed.zip file.

2

Import Creator Kit Solution

  • Sign in to make.powerapps.com
  • Select your target environment
  • Go to Solutions → Import Solution
  • Browse, select the Creator Kit zip, and import
3

Wait for Completion

Do not close the browser during import. Wait for all validation, extraction, and deployment steps to complete successfully before proceeding.

Optional Requirements

Optional
AI Builder Credits
~50 credits/Generative Answers test; ~120 credits/Prompt Advisor iteration.
Optional
Azure Application Insights
For additional AI-generated answer telemetry details.
Optional
Power BI Pro / PPU
Required to publish and embed the Conversation KPIs dashboard.
Optional
Power Platform Admin Role
For full Agent Inventory tenant visibility across all environments.
Optional
SharePoint Access
For SharePoint synchronization feature to agent knowledge bases.

Installation Method A: AppSource / Marketplace

▼
⭐
Recommended Method AppSource is the easiest path. Microsoft validates the package, the installer handles connection prompts, and you get automatic update notifications. Choose this unless you need a specific version.

Deploy from AppSource

1

Navigate to AppSource

Go to https://appsource.microsoft.com and search for "Copilot Studio Kit", or use the direct alias: https://aka.ms/CopilotStudioKit

2

Get It Now

Select Get it now on the Copilot Studio Kit listing page.

3

Sign In

Sign in with your Microsoft work or school account if prompted.

4

Select Environment

In the Install view, use the drop-down to select your target environment.

  • Ensure Dataverse is enabled in this environment
  • Ensure Creator Kit is already installed
  • Ensure PCF components are enabled
5

Accept Terms & Install

Review the Terms and Statements. If you agree, check the confirmation boxes and select Install.

6

Wait for Installation (10–20 min)

Do not close the browser. You will receive an email when installation completes. Monitor progress in the Power Platform Admin Center under Environments → [Your Env] → Solutions.

Post-Deployment Steps (AppSource)

  1. Sign in to make.powerapps.com
  2. Go to Solutions
  3. Select Default Solution
  4. Select Connection references
  5. Locate Copilot Studio Kit - Dataverse and open for editing
  6. Ensure all required fields are filled and connection is valid. Create new connection to Dataverse if needed.
  7. Select Save
  8. Proceed to Section 4: Post-Installation Configuration
3

Installation Method B: GitHub

▼
📖
Use GitHub When You need a specific version, want to install without AppSource, need to use the optional AgentInventoryUsage solution, or are performing a controlled upgrade.

Download & Install

1

Download Latest Release

Navigate to: https://github.com/microsoft/Power-CAT-Copilot-Studio-Kit/releases/latest

Download CopilotStudioKit_[version]_managed.zip to an accessible local folder.

2

Sign In & Select Environment

Go to https://make.powerapps.com and select your target environment from the environment picker (top right).

3

Open Solutions & Import

In the left navigation, select Solutions. Then select Import solution from the toolbar.

4

Browse & Select File

Select Browse, navigate to the downloaded CopilotStudioKit_managed.zip file, and select it. Click Next.

5

Review & Configure Connections

After reviewing solution info, select Next. If prompted, create required connections (Dataverse, SharePoint, etc.). Select Import.

⚠
ImportantLeave "Conversation KPIs report" values as-is. You will configure these in a later step.
6

Wait for Import to Complete

The import may take 10–20 minutes. Do not close the browser tab. You will see a success banner when complete.

Upgrading an Existing Installation

  1. Download the latest CopilotStudioKit_managed.zip from GitHub Releases
  2. Sign in to make.powerapps.com and select your environment
  3. Go to Solutions → Import Solution → Browse and select the new zip
  4. Select Next — Power Apps recognizes this as an upgrade to the existing managed solution
  5. Confirm upgrade and select Next, leave KPI values as-is, then Import
🚨
Known Upgrade Issue Error: "Solution 'Power CAT Copilot Studio Kit' failed to import: ImportAsHolding failed: Cannot delete attribute: cat_copilottestrunid from Entity: cat_CopilotTestRun since the attribute is not a custom field."

Workaround: Stage the upgrade before applying it. See: https://learn.microsoft.com/en-us/power-platform/alm/update-solutions
🔔
Stay Up To Date Subscribe to GitHub releases: Repository page → Watch → Custom → Releases → Apply. You'll get email notifications for every monthly release.

Post-Installation Configuration

▼
ℹ
Applies to Both Methods Complete these steps after installing via either AppSource or GitHub. Use the Setup Wizard (Section 5) to automate most of these steps.

Configure Connection References REQUIRED

Connection references link the Kit's cloud flows to actual connectors in your environment.

Connection Reference Purpose Required?
Copilot Studio Kit - Dataverse Core Kit data access — test automation, inventory, most features Required
Copilot Studio Kit - SharePoint Source SharePoint site for content synchronization Optional
Copilot Studio Kit - Power Platform for Admins Agent Inventory: enumerate all environments in tenant Optional

Steps:

  1. Sign in to make.powerapps.com
  2. Go to Solutions → Default Solution
  3. Select Connection references
  4. Open each connection reference listed above for editing
  5. Ensure all required fields are filled and the connection is valid
  6. Create a new connection if needed, then Save

Configure SharePoint Synchronization Optional

Required only if you plan to synchronize content from SharePoint to Copilot Studio knowledge bases.

  1. Configure the Copilot Studio Kit - SharePoint connection reference to point to the source SharePoint site
  2. Ensure the Dataverse URL environment variable points to the Dataverse instance of the target agents
  3. Configure this at any time before using the SharePoint Sync feature
⚠
Flow Dependency Do NOT enable SharePoint Synchronization cloud flows until the SharePoint connection reference is fully configured. Flows that start with "SharePoint Synchronization" should remain Off until then.

Configure Agent Inventory Optional

Enables tenant-wide visibility into all Copilot Studio agents.

🏢
Recommendation For full tenant-wide monitoring, configure both connection references below with accounts that have the highest permitted access levels.
  1. Configure Copilot Studio Kit - Power Platform for Admins with an account holding Power Platform Administrator role — provides full environment enumeration across all tenants
  2. Configure Copilot Studio Kit - Dataverse with an account holding System Administrator access to all target environments — gathers agent details from each environment
  3. Accounts with limited permissions will see only the environments they have direct System Admin access to

Enable Cloud Flows REQUIRED

⚠
Critical Order Enable flows in this exact order: (1) Flows ending in "Grandchild", (2) Flows ending in "Child", (3) All remaining flows. Incorrect order can cause dependency failures.
  1. Navigate to https://make.powerautomate.com
  2. Ensure the correct environment is selected
  3. Select Solutions → Copilot Studio Kit
  4. Select Cloud flows
  5. Review Status column — all required flows should show On
  6. Enable Grandchild flows first
  7. Enable Child flows next
  8. Enable all remaining flows for features you plan to use
ℹ
Selective Enablement You only need to enable flows for features you plan to use. Flows are named with the feature as a prefix, making it easy to identify which flows correspond to which features.

Configure Conversation KPI Dashboard Optional

Required only if you want the embedded Power BI Conversation KPIs dashboard in the app.

  1. Download Conversation.KPIs.zip from the GitHub repository
  2. Extract and open Conversation KPIs.pbit in Power BI Desktop
  3. Enter the Dataverse URL as "Environment URL" (e.g., org3c23948.crm.dynamics.com — no https:// prefix)
  4. Refresh data and sign in when prompted
  5. Save and publish to a Power BI workspace. Note: Report Name + Workspace Name
  6. Open in Power BI Service and note: Workspace GUID + Report GUID from the URL
  7. Verify data source credentials: ... → View Semantic Model → File → Settings
  8. Set a data refresh schedule
  9. In Power Apps: Solutions → Default Solution → Environment variables → Conversation KPIs Report
  10. Replace placeholders in Default Value with your Workspace Name, Workspace GUID, Report Name, Report GUID
  11. Select Save
📌
My Workspace GUID If publishing to "My Workspace" in Power BI, use this fixed GUID: 00000000-0000-0000-0000-000000000000

Set Up Compliance Hub Optional

⚠
Prerequisite Compliance Hub requires Agent Inventory to be fully configured first. Complete the Agent Inventory tab before setting up Compliance Hub.

Follow the detailed Compliance Hub setup guide:

https://github.com/microsoft/Power-CAT-Copilot-Studio-Kit/blob/main/COMPLIANCE_HUB.md

Compliance Hub allows you to:

  • Define risk thresholds and policies for agent configurations
  • Automatically create compliance cases when violations are detected
  • Track cases through SLA-driven review lifecycles
  • Send Teams/Outlook notifications to agent makers for remediation
  • Configure enforcement actions: manual review, quarantine, or delete

Using the Setup Wizard

▼
🪄
Available Since May 2025 The Setup Wizard provides an in-app guided experience for most post-deployment configuration steps. Use the Setup Wizard as your primary configuration method — it's faster, validates as you go, and reduces manual errors.

Access the Setup Wizard from the Home page of the Copilot Studio Kit app after deployment.

✅ Prerequisites Check

Validates environment dependencies and settings before you begin configuration. Catches issues before they cause problems.

🔗 Connection References

Lists all connection references, shows current status, and allows inline editing and validation without leaving the app.

⚙️ Environment Variables

Displays all environment variables with descriptions, usage guidance, and direct editing. Eliminates guesswork.

🔄 Flow Activation

Shows current on/off status of all flows and enables them in correct dependency order with a single click.

⚠
Not Covered by Setup Wizard These must still be done manually:
  • Publishing the Conversation KPIs Power BI report and scheduling data refresh
  • Creating Azure AD App Registrations for test automation (authentication + App Insights)

Security Roles & User Setup

▼

The Copilot Studio Kit ships with two current security roles. Assign users based on their responsibilities.

RoleIntended ForAccess Level
CSK - Administrator Solution administrators and governance owners who configure and maintain the kit Organization-level access to all CSK tables and features
CSK - Maker Developers and testers who build and validate Copilot Studio agents User-level access; can create/run tests, limited read on shared config
🚫
Deprecated Roles — Do Not Use The following legacy roles have been deprecated: "Copilot Studio Kit - Administrator", "Copilot Studio Kit - Configurator", "Copilot Studio Kit - Tester/KPI Viewer". Migrate all users to the new CSK - Administrator or CSK - Maker roles.

Accessing the App

▼
1

Sign In to Power Apps

Navigate to https://make.powerapps.com

2

Select Environment

Use the environment picker (top right) to select the environment where you installed Copilot Studio Kit.

3

Navigate to Apps

Select Apps in the left navigation panel.

4

Launch the App

Locate and select Copilot Studio Kit. Select Play to launch.

5

Run Setup Wizard

From the app Home page, launch the Setup Wizard to verify and complete any remaining configuration.

6

(Optional) Bookmark the App

Bookmark the app URL for quick access. Share the URL with team members who have been assigned CSK security roles.

🚀
Next Steps After Launch
  1. Run the Setup Wizard from Home to verify all configuration
  2. Configure your first Copilot Studio agent in the Kit
  3. Create a test set and run your first automated test
  4. Review Agent Inventory for tenant-wide visibility
  5. Configure Compliance Hub policies for governance

Known Issues & Troubleshooting

▼

Click each issue to expand details and resolution steps.

🚨 Upgrade Error: cat_copilottestrunid Cannot Delete Attribute ▼

Symptom: "Solution 'Power CAT Copilot Studio Kit' failed to import: ImportAsHolding failed with exception: Cannot delete attribute: cat_copilottestrunid from Entity: cat_CopilotTestRun since the attribute is not a custom field."

Resolution: Stage the upgrade before applying it. This is a known issue under investigation by Microsoft.
See: https://learn.microsoft.com/en-us/power-platform/alm/update-solutions

🚫 Dependency Error: Creator Kit Not Installed ▼

Symptom: Error during solution import indicating missing components or dependencies.

Resolution: Install Creator Kit Core first from: https://github.com/microsoft/powercat-creator-kit/releases

🚫 PCF Components Not Enabled ▼

Symptom: Components fail to render, or solution import fails with component errors.

Resolution: Enable PCF via: Power Platform Admin Center → Environments → [Your Env] → Settings → Features → "Power Apps component framework for canvas apps" → On.

⚠ Connection Popup on First App Launch ▼

Symptom: Users see a popup asking to allow connections when first opening the app.

Resolution: This is expected behavior. Users should review and allow the listed connections. See ALLOW_CONNECTION_POPUP.md in the GitHub repo for detailed guidance.

⚠ Cloud Flows Remain Off After Import ▼

Symptom: Some or all cloud flows show Status = Off after solution import.

Resolution: Manually enable flows in Power Automate in the correct order: Grandchild flows first, then Child flows, then all others. Use the Setup Wizard for guided activation.

ℹ Agent Inventory Shows Limited Environments ▼

Symptom: Agent Inventory only shows a subset of tenant environments.

Resolution: The Power Platform for Admins connection reference needs a Power Platform Administrator account for full visibility. The Dataverse connection needs System Admin access to all target environments.

⚠ AI Builder Credits Exhausted ▼

Symptom: Test automation (Generative Answers) or Prompt Advisor returns errors or fails.

Resolution: Check AI Builder credit balance in Power Platform Admin Center. Budget approximately 50 credits per Generative Answers test case and 120 credits per Prompt Advisor iteration. Purchase additional capacity as needed.

ℹ Custom Agent Authentication Limitations ▼

Known Limitation: Custom agent end-user authentication support in test automation is limited to Entra ID V2 with SSO.

Workaround: Design test cases with this limitation in mind. See ENABLE-AUTHENTICATION.md in the GitHub repo for setup guidance.
